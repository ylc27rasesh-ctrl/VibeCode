PART ONE · CHAPTER 01

What is vibe coding, really?

Vibe coding is a term coined in February 2025 by Andrej Karpathy (a co-founder of OpenAI and former AI lead at Tesla). The idea: instead of typing every line of code yourself, you describe what you want in plain English, and an AI model writes the code for you. You read it, run it, see what breaks, and tell the AI to fix or change things. You "surrender to the vibes" of the model.

A year later, in 2026, this is no longer a niche thing. It is the dominant way solo builders, small teams, and even working professionals create software. The tools have gotten genuinely good. Real apps with real users get shipped this way every day.

There are two big categories of vibe coding tools, and the difference matters a lot:

Category A
App Builders

You chat. They build the whole thing — frontend, backend, database, deployment. You may never see the code.

Lovable · Bolt · Replit · v0 · Hostinger Horizons · Base44
Category B
AI Coding Agents

They live on your computer (or terminal). They write and edit real code files, run commands, and often work alongside your editor.

Claude Code · Codex · Cursor · Antigravity · Windsurf · Cowork

Category A is where you'll start. Category B is where you'll graduate to when you want more control. By the end of this guide you'll be comfortable in both.

Why this guide exists

Most "vibe coding" content assumes you already know what a terminal is, what GitHub does, or what "deploying" means. This one doesn't. We start from zero and don't skip steps.

CHAPTER 02

The mental model of an app

Before any tools, you need a picture of what an app actually is. Almost every modern app has the same three pieces. Once you see them, every tool in this guide will make sense.

Frontend
What the user sees and clicks. The buttons, screens, colors, layout.
↔
Backend
The brain. Handles logic, logins, payments, calling AI models.
↔
Database
Where data lives — user accounts, posts, scores, settings.

Plus two helpers everyone needs:

  • Hosting — the computer on the internet that runs your app 24/7 so people can visit it.
  • Domain name — the friendly address like mycoolapp.com that points to your hosting.

A "full-stack" app means all three layers exist. An AI agent is usually just a backend that talks to an AI model and maybe other services like Slack or your email. A game can sometimes be just a frontend if it runs entirely in the browser.

The good news: modern vibe coding tools handle all of this for you when you're starting out. Lovable, Bolt, and Replit will spin up frontend + backend + database + hosting with one prompt. Your job is just to describe the thing.

CHAPTER 03

The plain-English glossary

Keep this section bookmarked. Every term here will show up across the rest of the guide and across every tool you use.

Code
Instructions for a computer, written in a programming language. Looks like const greeting = "hello". You won't write much of it; AI will.
File & folder
A project is just a folder of text files. index.html, app.js, style.css — these are typical file names. The folder is sometimes called a "repo" (short for repository).
Markdown
A simple way to format text using symbols. **bold** becomes bold. # Heading becomes a heading. Files end in .md. You'll see Markdown everywhere — README files on GitHub, AI agent instructions, Antigravity's "artifacts." It's easier to read than HTML.
Terminal / Command Line
A black window where you type text commands instead of clicking. On Windows it's called PowerShell or Command Prompt. Don't be scared of it — vibe coding tools have made it mostly optional.
VS Code
Visual Studio Code. Microsoft's free code editor. It's basically a souped-up Notepad — a window for viewing and editing the files in your project. It is the most popular code editor on the planet. Many AI coding tools are built on top of it.
IDE
Integrated Development Environment. Fancy phrase meaning "a code editor with extra stuff built in" (a terminal, a file browser, debugging tools). VS Code is the most famous one. Cursor, Antigravity, and Windsurf are all IDEs built on VS Code.
Node.js
A program that lets your computer run JavaScript outside of a web browser. Many AI coding tools need Node.js installed because they're written in JavaScript. You install it once and forget about it.
npm / npx
Comes with Node.js. npm ("Node Package Manager") installs little JavaScript programs called "packages." npx runs a package without installing it permanently. You'll type npx antigravity-ide to launch Antigravity, for example.
Git
A program that tracks every change you make to your files, like a time machine. If you break something, you can roll back. Created by Linus Torvalds (the guy who made Linux). You'll install it once.
GitHub
A website that stores your Git-tracked projects in the cloud. Free for most uses. Think of it as Google Drive, but designed for code. Lets you back up your work and collaborate. Git is the tool. GitHub is the website.
Repository (Repo)
A single project on GitHub. Just a folder of files with version history.
Commit
A snapshot of your project at one moment in time, with a message describing what changed. You "commit" your changes so Git remembers them.
Push / Pull
Push = upload your commits to GitHub. Pull = download new changes from GitHub to your computer.
API
Application Programming Interface. A way for one program to talk to another. When your app "calls the OpenAI API," it's sending a request over the internet to OpenAI's servers and getting an answer back.
API key
A long secret password that proves you're allowed to use an API. Treat it like a credit card number — never paste it into a public place.
Frontend
The visible part of your app — what runs in the user's browser or phone. Built with HTML (structure), CSS (style), and JavaScript (behavior).
Backend
The invisible server part — handles logins, stores data, does heavy lifting. Often written in JavaScript (Node), Python, or similar.
Database
Organized storage for your app's data. The most common type for beginners is PostgreSQL via Supabase.
Supabase
A free/cheap service that gives you a database, user authentication (logins), and file storage with almost no setup. Lovable, Bolt, Antigravity, and others connect to it directly.
Deployment / Hosting
Putting your app on a public computer so the world can use it. Common free hosts: Vercel, Netlify, Cloudflare Pages, Render.
Framework
A pre-made skeleton of code that handles common patterns so you don't reinvent the wheel. React is a framework for building frontends. Next.js wraps React with more features. You don't need to learn these — AI will pick one for you.
Library / Package
A reusable chunk of code someone else wrote that you "import" into your project. react, lodash, tailwind are all libraries.
Token (in AI tools)
A rough chunk of text — about ¾ of a word. AI tools charge by tokens used. When Bolt or Lovable say "you have 5M tokens," they mean roughly 5 million words of conversation total.
Context window
How much text an AI can hold in its head at once. Bigger = it can read more of your code before answering. Claude Opus 4.7's context window is huge by 2026 standards.
Prompt
The instruction you give the AI. "Build me a to-do app with dark mode" is a prompt.
Agent
An AI that doesn't just answer questions — it takes actions. Reads files, runs commands, browses the web, edits code on its own. Antigravity, Claude Code, Cowork, and Codex are all agentic.
MCP (Model Context Protocol)
An open standard, started by Anthropic, for letting AI tools connect to outside services (Slack, Gmail, Notion, Supabase). When you hear "I added an MCP server," it means hooking the AI up to a new tool.
Artifact
A file the AI produces as proof of its work — a plan, a summary, a test result. Antigravity and Claude both use this term.
Sandbox / VM
An isolated mini-computer the AI runs commands inside, so it can't accidentally trash your real machine. Cowork runs in a VM (virtual machine).
PART TWO · CHAPTER 04

Installing your starter stack on Windows

You only need to do this once. After today, all of these are just background scenery. Total time: maybe 30 minutes.

Install Node.js

Go to nodejs.org and download the "LTS" version (Long Term Support — the stable one). Run the installer. Click "Next" through everything. When asked about "automatically install necessary tools," leave it checked. This installs Node, npm, and npx all at once.

Verify it worked: Press the Windows key, type powershell, hit Enter. In the blue window, type node --version and press Enter. If you see something like v20.11.0, you're done.

Install Git

Go to git-scm.com/download/win. Download. Run the installer. Click "Next" on everything — defaults are fine. When asked about the default editor, you can pick "Use Visual Studio Code" if it's already installed, or just leave it on the default.

Verify: In PowerShell, type git --version. You should see something like git version 2.43.0.

Install VS Code

Go to code.visualstudio.com. Download for Windows. Install. During setup, tick the boxes for "Add 'Open with Code' action" and "Add to PATH" — these make life easier later.

Create a GitHub account

Go to github.com and sign up. Pick a username you wouldn't mind colleagues seeing — your GitHub profile is sometimes shared as a portfolio. Free is fine forever.

Connect Git to GitHub

In PowerShell, run these two commands (substitute your real name and the email you used for GitHub):

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

This tells Git who you are. When you push to GitHub for the first time, a browser window will pop up asking you to sign in — do that and it'll remember you forever.

Install Claude Desktop

Go to claude.com/download and grab the Windows installer. Sign in with your Anthropic account. This single app contains Chat, Cowork, and Code — three of the most important tools in this guide. You'll need a paid plan (Pro from $20/mo or Max for heavier use) for Cowork and Code.

(Optional) Sign up for one app-builder

Pick one of Lovable, Bolt.new, or Replit. All have free tiers. We'll use one of these later for your first project. If you can't decide: Lovable. It's the gentlest on-ramp.

A note on PowerShell

If a command in this guide says "run X in your terminal," it means: open PowerShell, paste the command, press Enter. That's it. You can also right-click any folder in Windows Explorer and choose "Open in Terminal" to start in that folder.

CHAPTER 05

Understanding GitHub without the headache

GitHub is intimidating from the outside. From the inside, here's all it actually is: a website that stores your project folders and tracks every change. That's the whole thing. The fancy vocabulary is just labels for simple actions.

The five actions you'll actually use

ActionWhat it doesWhen
Clone Download an existing project from GitHub to your computer Starting work on something that already exists
Commit Save a snapshot of your current work with a description Every time you finish a chunk of work — minimum once a day
Push Upload your commits to GitHub After committing, so the cloud has the latest
Pull Download new commits from GitHub Working from a second computer, or after a collaborator changed things
Branch Create a parallel version of your project to try something risky Big experiments. Skip until you need it.

Why this matters for vibe coding

Three real reasons:

  1. Backup. If your laptop dies, your code is safe on GitHub.
  2. Undo. When the AI breaks something subtle, you can roll back to your last commit and try a different approach.
  3. Portability. Tools like Lovable, Bolt, and Cursor all read from and write to GitHub. You can build something in Lovable, push it to GitHub, then keep editing it in Cursor. GitHub is the universal handoff format.
The one habit to develop

Commit and push before you ask the AI to make a big change. If the AI butchers it, one click rolls everything back. This single habit will save you from 90% of vibe coding disasters.

Your first repo — in 60 seconds

  1. Go to github.com, click the green "New" button.
  2. Name it (e.g. my-first-app), leave everything else default, click "Create."
  3. On the next page, GitHub shows you the URL of your new repo. Copy it.
  4. In PowerShell, navigate to where you want it (e.g. cd Desktop) and run: 
    git clone https://github.com/YOUR_USERNAME/my-first-app.git
cd my-first-app
  5. You now have an empty project folder, ready to fill with code.

To commit and push later:

git add .
git commit -m "Added the login page"
git push

git add . means "stage all the changes." commit takes the snapshot. push uploads it. You'll do this thousands of times.

CHAPTER 06

VS Code, the universal canvas

Open VS Code. The first time it looks intimidating; after a week, it'll feel like Notepad. Here's the only layout you need to know:

  • Left sidebar (icons) — switch between Files, Search, Source Control (Git), Extensions, and a Run/Debug button.
  • Explorer panel — the file tree of your project. Click File → Open Folder to load a project.
  • Editor — where you read and edit code.
  • Bottom panel — toggleable terminal. Press Ctrl + ` (backtick) to open/close it. This is your PowerShell, but built into the app.

The five extensions worth installing

Click the Extensions icon on the left sidebar, search for these, install:

Claude Code

The official Anthropic extension. Adds an AI pair-programmer right inside VS Code. Free with a paid Claude plan.

GitHub Copilot

AI auto-complete from GitHub. Free for students with GitHub Education. Suggests code as you type.

GitLens

Adds beautiful Git history right next to each line of code. You'll see who changed what, when, and why.

Prettier

Auto-formats your code so it looks tidy. Set it to run on save. AI tools sometimes write messy code; Prettier cleans it.

Live Server

One-click preview for plain HTML/CSS/JS projects. Right-click your index.html, "Open with Live Server," boom — it's live in your browser.

You will hardly ever type code in VS Code by hand. But you'll use it constantly to look at what the AI made, run it, and commit it to Git.

The recommended journey

Don't try to use them all. Here's the path I'd suggest for your goals (web apps with logins/databases + AI agents + games):

Build your first working web app entirely by chatting. No setup. See an idea become a real thing live on the internet. Get the rush.

Week 2 — GitHub + VS Code

Switch to a full agentic IDE. Build something more ambitious from scratch. Run multiple agents at once.

Week 7+ — Mix and match

Use Lovable for the first draft, GitHub to move it, Antigravity for serious editing, Codex for background tasks. This is how working vibe coders operate in 2026.

CHAPTER 08

Claude Cowork

Cowork is Anthropic's "Claude Code for non-coders." It lives inside the Claude Desktop app (the one you installed in Chapter 4) as a tab next to Chat and Code. You point it at a folder on your computer, describe an outcome — "organize my Downloads, file receipts in a folder by month, summarize all PDFs in here" — and Cowork plans the steps, runs them in a sandboxed mini-computer on your machine, and delivers finished files.

Why is this in a vibe coding guide if it's "non-coding"? Two reasons:

  1. Cowork can absolutely build small apps and scripts. It just doesn't lead with that framing.
  2. Learning to delegate to an agent in Cowork is the perfect on-ramp to delegating to one in Claude Code, Codex, or Antigravity. The shape of the conversation is the same.

What Cowork is good at

  • File organization — "sort my Downloads folder by type and date"
  • Research compilation — "read these 15 PDFs and write me a memo with the key insights"
  • Document drafting — "use these source files to draft a quarterly report in Word"
  • Data extraction — "pull the totals from these receipt screenshots into a spreadsheet"
  • Scheduled tasks — type /schedule and Cowork will run a job on a recurring schedule. Daily news briefings, weekly summaries.
  • Computer use — Cowork can take over your mouse and keyboard to operate apps that don't have direct integrations (in research preview, with permission per app).

Your first Cowork session

Open Claude Desktop, click the Cowork tab

You'll see three tabs at the top: Chat, Cowork, Code. Cowork is in the middle.

Tick "Work in a Folder" and pick one

Start with something low-stakes — a folder of receipts, a Downloads cleanup, a sample data spreadsheet. Cowork will ask permission to read, edit, and (with confirmation) delete files in that location.

Describe the outcome

Not the steps. Not the tools. The outcome. Example: "Look at all files in this folder. Group them into subfolders by year-month based on each file's date. For PDFs with no clear date, open them and use the date inside the document."

Review the plan, then approve

Cowork shows you what it intends to do before doing it. Read it. If it looks right, let it run. If not, tell it what to do differently.

Walk away

Cowork keeps working as long as the Claude Desktop app is open. You can also message it from your phone (Claude mobile app) and pick up where you left off. Just don't close the app or let your computer sleep mid-task.

Safety note

Cowork can delete files. It always asks before doing so, but real deletes are real. Always back up the folder you're letting it work in, or test on a copy first. Same for "computer use" — start with apps you trust, watch the first few sessions, never use it for banking or medical records.

CHAPTER 09

Claude Code

Claude Code is the coding-focused sibling of Cowork. Same brain, but specialized for working with code projects. It comes in three flavors:

  • Desktop tab — inside the Claude Desktop app, alongside Chat and Cowork. The friendliest entry point.
  • CLI — a command-line tool you run in PowerShell. The power-user experience. Skip for now.
  • VS Code extension — Claude Code embedded inside VS Code itself. Best when you want to keep using your existing editor.

How Code differs from Cowork

CoworkClaude Code
Mental model Knowledge worker delegating tasks Pair-programmer editing code with you
Permissions "Work in a Folder" + per-app approval Reviews each file edit with a diff before applying
Environment Sandboxed VM Your real files (or a cloud VM via "Remote")
Best for Docs, files, research, automation Building & modifying software

Your first Claude Code session

  1. Open Claude Desktop, click the Code tab.
  2. Pick "Local" and choose a project folder. Start with something tiny — even an empty folder is fine.
  3. It starts in Ask permissions mode: Claude proposes every change, shows you a "diff" (the lines being added in green, removed in red), and waits for you to click Accept or Reject. Stay in this mode for your first month.
  4. Try a prompt like: "Create a simple landing page with a headline, a paragraph of placeholder text, and a call-to-action button. Use Tailwind CSS for styling. Make it mobile-friendly."
  5. Watch what it does. Approve each change. Read the diff. Don't worry if you don't understand the code yet — you're getting fluent through exposure.
CLAUDE.md — the magic file

Create a file called CLAUDE.md in your project root. Anything you write inside it becomes Claude Code's standing instructions for that project. Examples: "Always use TypeScript, not plain JavaScript. Don't write tests unless asked. Comments should be brief." This single file dramatically improves consistency across long sessions.

CHAPTER 10

Google Antigravity

Antigravity is Google's agentic IDE, launched late 2025 and now free to use. It is a fork of VS Code — meaning it looks and feels identical to VS Code, but with deep AI integration baked in from the ground up. Powered by Gemini 3 Pro, with optional Claude Sonnet 4.5 and GPT-OSS models. Its big differentiator: multiple agents working in parallel, with a built-in browser they can drive themselves.

What makes Antigravity unusual

  • Agent Manager — a dashboard where you can spin up several agents on the same project. One researches an API while another builds the UI while a third writes documentation.
  • Built-in Chrome browser — Antigravity has a real Chrome window inside the IDE. Agents can click around your app like a user would, take screenshots, and verify their work actually behaves correctly.
  • Artifacts — as agents work, they produce Markdown files documenting their plan, progress, and final summary. You can read these to follow along or audit later.
  • Free tier is generous — at the time of writing, fully free. Pro/Ultra tiers exist for higher rate limits.
  • Imports from VS Code — your existing settings, extensions, keybindings all carry over.

Getting started with Antigravity

Download from antigravity.google

Pick the Windows installer. Run it. Sign in with your Google account.

Pick your mode at first launch

Antigravity will ask if you want agent-first, AI-assisted, or traditional. As a beginner, choose agent-first. This puts the Agent Manager front and center.

Create your first project

File → Open Folder → create a new folder somewhere (e.g. C:\Users\You\my-antigrav-app). Antigravity will open it as an empty project.

Give a high-level goal

In the agent chat: "Build a habit tracker web app. Users can sign up, add daily habits with custom icons, and see a calendar view with streaks. Use React, Tailwind, and Supabase. Set up everything from scratch." Then approve the agent's plan.

Watch the multi-agent magic (or just one agent for a small task)

For complex builds, click "Add agent" in the Agent Manager. Give the second agent a parallel job: "Design a logo for the habit tracker." Both run side by side.

The .agent folder and GEMINI.md

Antigravity adds a hidden folder called .agent to your project. Inside are rules, skills, and workflows that customize how the agents behave. There's also a GEMINI.md file (their version of CLAUDE.md) for project-level standing instructions. Don't touch these on day one. Just know they exist; on day thirty you'll be editing them to teach agents how your project works.

Antigravity vs Cursor in one line

Cursor optimizes for the developer typing alongside the AI. Antigravity optimizes for the developer delegating to multiple AIs at once. If you like supervising rather than collaborating moment-to-moment, Antigravity is the better fit.

CHAPTER 11

OpenAI Codex

Codex started life in May 2025 as OpenAI's cloud coding agent, accessed through ChatGPT. As of 2026 it's an entire suite — desktop app (Mac and Windows), CLI, an IDE extension that hooks into VS Code, and a web interface — all sharing the same agent and connected through your ChatGPT account. The latest model is GPT-5.3-Codex, optimized specifically for coding and computer-use tasks.

What Codex does that's different

  • Parallel agents in worktrees — fire off 3, 5, even 10 tasks at once. Each runs in an isolated copy of your code. You review the diffs that succeed.
  • Background tasks — give Codex a task in the morning before coffee, come back to a finished pull request. The "morning queue" workflow is hugely popular among working developers.
  • In-app browser — like Antigravity, Codex now has a browser. You can leave comments directly on a rendered web page to tell the agent "this button should be bigger and live at the top right."
  • Memory — Codex now remembers your preferences across sessions: which libraries you like, how you name things, where your dev server runs.
  • 90+ plugins — Atlassian (Jira), GitHub, GitLab, Figma, Linear, Vercel, Cloudflare, Render, Netlify. Codex can deploy your app to any of these.
  • Skills — bundles of instructions that teach Codex how to do specific things. Want it to build Figma designs into pixel-perfect React? Install the Figma skill.

How to start with Codex

  1. Get a ChatGPT subscription (Plus, Pro, Business, or Enterprise). Free and Go users currently get limited Codex access too.
  2. Install the Codex desktop app from openai.com/codex. Sign in with the same ChatGPT account.
  3. In the app, click "New project" and either point it at a local folder or a GitHub repo.
  4. Give it a task in plain English. "Add dark mode to this app. Use a toggle in the top right and remember the user's choice."
  5. Codex creates a worktree (an isolated copy), works on it for 1–30 minutes, and posts a diff. You review, accept, or comment.

When to reach for Codex over Antigravity

Honest answer: they overlap a lot. The differences in practice:

  • Pick Codex if you already pay for ChatGPT, want the strongest parallel-task workflow, and like the background queue model.
  • Pick Antigravity if you want a free option, prefer Google's Gemini 3 model, or want the integrated browser-driving agent to be a primary feature.
  • Use both if you can afford it — they coexist fine. Codex's CLI is invokable from any terminal, including Antigravity's built-in one.
CHAPTER 12

The app builders: Lovable, Bolt, Replit, v0

These four are the friendliest entry points for someone with no coding experience. You don't install anything — they all run in your browser. You chat, an app appears, you can publish it. Here's the honest breakdown of when to use which.

Lovable

Best for: non-technical founders building their first full-stack web app. Lovable is opinionated — it uses React + Tailwind + Supabase under the hood — which is actually a feature for beginners. Fewer choices means less to think about. Strong visual preview as you build, two-way GitHub sync, Stripe payment integration baked in.

Pricing: free tier with daily limits, paid plans start around $20/month.

Watch out: Lovable will happily build apps with weak database security if you don't ask for protection. Always include "use Supabase Row Level Security" in your initial prompt.

Bolt.new

Best for: quick prototypes, single-page apps, and now mobile apps too (Bolt added native mobile via Expo in late 2025). Two AI agents to choose from: Claude (more reliable) and v1 (faster/cheaper). Imports existing projects from Lovable, which is handy.

Pricing: token-based, can be opaque. Daily free allotment is enough to test the waters.

Watch out: token usage scales with how big your codebase has grown. The same edit costs more tokens at week 6 than week 1.

Replit

Best for: people who want everything in one place — coding, AI assistant, database, hosting, and even mobile preview. Replit's Agent generates complete runnable apps and handles deployment within the same browser tab. Supports 50+ programming languages.

Pricing: free tier (with a 30-day published-app expiry), $20/month Core, $40/month Teams.

Watch out: more technical complexity surfaces than in Lovable. You'll see real folder structures and may need to occasionally read code.

v0 (by Vercel)

Best for: people who want gorgeous frontends. v0 is exceptional at making UI but it doesn't really do backends — pair it with Supabase if you need data. Deployment to Vercel is one click.

Pricing: generous free tier, paid plans from $20/month.

Watch out: you'll need to add your own database/auth layer for anything beyond a static site. Not a one-stop shop.

How to pick today

If you want one place to build full-stack web apps with logins, databases, and payments, pick Lovable. For your specific goals (web apps with logins/databases + games + mobile versions), Lovable is the most direct path. You can always migrate later.

A pro pattern

Start in Lovable until you hit a wall. Connect it to GitHub. Pull the code into VS Code (or Antigravity/Cursor). Continue from there with an agentic IDE. This is the single most common path for vibe coders going from "first app" to "shipping serious software."

CHAPTER 13

Cursor & Windsurf

These two are direct competitors and look almost identical: both are VS Code forks with AI-first design. You'd use them once you're past the "I just want it to build for me" phase and want tighter control over what the AI does.

Cursor

The most beloved AI IDE among working developers as of 2026. Three modes: Composer (multi-file edits via chat), Agent (autonomous changes), Edit (precise single-spot edits). Includes Bugbot for automated review and Cloud Background Agents for offloaded tasks. Free tier limited; Pro is $20/month.

Windsurf

Built around Cascade Flow — a single chat where the agent handles whole features end to end with previews and deploys. Has Rules and Memory features that personalize the agent over time. MCP integration is excellent. Free preview, paid tiers comparable to Cursor.

For your stage right now, you don't need either of these. Antigravity covers similar ground for free. Cursor and Windsurf become attractive once you're working on a real codebase and want the polished daily-driver experience that working developers swear by.

The build workflow

Every serious vibe coder follows roughly the same loop, whether they're using Lovable in a browser or Antigravity on a desktop. Learn this rhythm and you'll never feel lost again.

1. Spec
Write what you want in plain language.
→
2. Generate
Let the AI make a first version.
→
3. Run
See if it works. Click around.
4. Critique
Note what's wrong.
→
5. Iterate
Ask AI to fix one thing.
→
6. Commit
Save to Git. Loop back to 3.

The cardinal rule: small steps, frequent commits

The single biggest mistake beginners make is asking for too much at once. "Build me a complete clone of Instagram with stories, DMs, and reels" will produce a janky mess that's hard to fix. "Build a single page where users can sign up and log in" works beautifully.

After it works, commit. Then ask for the next slice: "Now add a page where logged-in users can upload a photo and see all their uploads." Commit again. Repeat.

Ten small commits beat one giant generation every time. When something breaks, you can roll back five minutes — not five days. The rule every vibe coder learns the hard way

The 80/20 of avoiding agent disasters

  1. Commit before any big ask. Always. (See above.)
  2. Stay in "Ask permissions" mode in Claude Code, Cursor, Antigravity. Watch the diffs. You'll learn what good code looks like by osmosis.
  3. If the AI is in a doom loop (trying the same fix five times, things getting worse), stop it. Roll back. Start a fresh conversation with a tighter prompt.
  4. When debugging, share the exact error message, not your interpretation of it.
  5. Periodically ask the AI to summarize what it's done. This forces it to re-read its own work and often catches drift.
CHAPTER 15

Prompting like a pro

Your prompts are the source code of vibe coding. A great prompt makes the difference between a thirty-second build and a thirty-minute mess. Here's the structure that consistently works:

The four-part prompt

Context: "I'm building a habit tracker for myself."
Goal: "Right now I need a sign-up and login page."
Constraints: "Use Supabase for auth. Email + password only. Tailwind for styles. Keep it minimal — one column, centered, no fancy animations."
Acceptance: "Done when I can sign up, log out, log back in, and the session persists across browser refreshes."

That single prompt will produce something usable on the first try about 80% of the time. Compare to: "make a login page lol" — which leaves the AI guessing about ten different things.

Eleven prompting tricks that punch above their weight

  • Show, don't just tell. "Make it look like Linear.app's homepage" beats "make it look modern."
  • Specify the stack. "Use React, Tailwind, Supabase, no other libraries." Removes guesswork.
  • Define the data shape. "Habits have a name, an icon, a color, and a daily log of timestamps." Prevents schema drift.
  • State what NOT to do. "Don't add comments. Don't write tests yet. Don't change the styling of any other page."
  • Ask for a plan first. "Before writing any code, describe in 3-5 bullets what you'll do and which files you'll touch."
  • Reference existing code. "Mimic the pattern from the Habits page when building the Goals page."
  • Tell it what good looks like. "Mobile-first. WCAG-AA contrast. Loads in under 2 seconds."
  • Use Markdown structure. Bullets and headings in your prompt make AI follow your instructions more reliably than a wall of text.
  • Be explicit about edge cases. "What if the user has no habits yet? Show an empty-state with a button to add the first one."
  • Ask for verification. "After you're done, walk through the user flow step by step and confirm each works." Antigravity and Codex will literally drive their built-in browser to do this.
  • End with the success criteria. "Don't mark the task complete until X, Y, Z all pass."

Three prompts to keep in your back pocket

When stuck

"Stop what you're doing. Read the entire codebase. Describe in plain English what this app does and how the pieces fit together. Then propose three different approaches to solve [problem]. Don't write code yet."

When the AI keeps making the same mistake

"You've tried [X] three times and it keeps failing because [Y]. Forget that approach entirely. What's a completely different way to achieve the goal?"

When you need to add a feature without breaking anything

"I want to add [feature]. First, list every file you'll need to change and why. Show me the list. Don't change anything until I approve. After I approve, make changes one file at a time and pause between files."

CHAPTER 17

Project: build an AI agent

An "AI agent" is a program that takes actions on your behalf — not just a chatbot, but something that reads emails, posts to Slack, summarizes documents, makes decisions. In 2026 these are easier to build than ever. Three approaches, ordered from easiest to most powerful:

Approach 1 — Use Cowork's scheduled tasks

The simplest "agent" is a recurring Cowork task. Open Cowork, type /schedule, and define a task like: "Every morning at 7am, read the news websites in my bookmarks folder, summarize the 5 most important AI stories into a Markdown file in Documents/Daily-Briefings/, and email it to me at my address."

Cowork will execute this on the schedule you set as long as your computer is on. No code. No prompts at run-time. Just a brief once.

Approach 2 — Build an agent app in Lovable or Bolt

When you want other people to use your agent (not just yourself), wrap it in an app. Example: build a Lovable app where users paste a YouTube URL, and your app uses an AI API to fetch the transcript, summarize it, and produce flashcards.

Sample prompt for Lovable

Build "Notetaker" — a tool where users paste a YouTube link and get back AI-generated flashcards. Use Supabase auth. On the home page, a single input for the URL and a "Generate" button. When clicked, fetch the transcript using the youtube-transcript library, send it to the OpenAI API (I'll add my key in environment variables) with a prompt asking for 10 flashcards in JSON, then display them as flippable cards. Save them to the user's account so they can revisit. Use Tailwind, minimal design, dark mode by default.

The key bit here is the AI API key. You'll sign up at platform.openai.com or console.anthropic.com, generate a key, and add it to your Lovable project's Environment Variables. Treat this key like a credit card.

Approach 3 — Multi-step agents in Claude Code or Antigravity

For agents that need to do multiple things in sequence (read files, browse the web, write reports, send results), the agentic IDEs shine. You're essentially building an agent as a project file. A common pattern:

  1. Create a folder. Open in Claude Code.
  2. Tell it: "Build a Python script that takes a list of competitor URLs from a CSV, visits each one, extracts their pricing pages, and outputs a comparison spreadsheet."
  3. Claude writes the script, runs it on test data, fixes errors itself, and shows you results.
  4. You schedule it to run weekly using Windows Task Scheduler, or just trigger it manually when needed.
MCP — the magic word for serious agents

The Model Context Protocol (MCP), open standard by Anthropic, lets your AI tools plug into Slack, Notion, Gmail, your CRM, and more. Both Cowork and Claude Code and Codex and Antigravity speak MCP. When building agents, ask the AI to "use the Slack MCP" or "use the Notion MCP" — it'll know what to do.

CHAPTER 19

Mobile versions of your web apps

You asked specifically about mobile versions of your web apps. There are three real options in 2026, in increasing order of effort:

Option 1 — Responsive design (free, instant)

Modern app builders produce responsive web apps by default. They auto-adapt to phone screens. Users visit your URL on their phone, it works. Done. For 80% of apps this is what you should ship first.

To make sure responsiveness is solid, every time you finish a feature, prompt: "Test this on a 375px-wide viewport. Fix any layout issues. The whole interface should be thumb-reachable."

Option 2 — PWA (Progressive Web App)

A PWA is a web app that users can "install" to their home screen like a native app. It can work offline and send notifications. Almost no code changes — just adding a manifest file and a service worker.

Prompt: "Turn this app into a PWA. Add a manifest with icons, enable installability, and cache the main pages for offline use."

Option 3 — Native mobile apps via Expo

When you truly need an app in the App Store or Play Store, the modern path is Expo + React Native. Bolt.new can produce these directly now. Lovable can too with the right prompt. The same React skills the AI is using for your web app translate.

Be warned: shipping to the App Store involves Apple's developer program ($99/year), code-signing certificates, and review delays. Save this for when you have an app that genuinely needs to be in the store.

CHAPTER 21

Debugging when things break

Things will break. The frequency drops a lot once you internalize "small steps, frequent commits," but you'll never get to zero. Here's the order of operations when something goes wrong:

  1. Reproduce it. Make it happen again on purpose. Note the exact steps.
  2. Capture the error. If your browser console (F12 → Console) shows a red error, copy the entire thing.
  3. Paste it into your AI tool verbatim, with: "This error appears when I click [X]. The full error: [paste]. The code where this happens is in [file]. Please investigate before changing anything."
  4. If it doesn't fix it in two attempts, roll back to your last working commit (git reset --hard HEAD~1 in PowerShell, or use VS Code's Source Control panel).
  5. Try a different approach. Start a fresh AI session. Don't carry the broken context forward.

The browser DevTools every vibe coder should know

Press F12 in any browser. You'll see panels:

  • Elements — inspect any HTML element and see its styles. Right-click anything → Inspect.
  • Console — see error messages and logs. Most JavaScript bugs show up here.
  • Network — see every request your app makes to its backend. Red entries = failed requests.
  • Application — see what's saved in cookies, localStorage. Useful for "why isn't the user staying logged in?" bugs.

When you ask AI to debug, say "Here's what the browser console shows: [paste]. Here's the failed request from the Network tab: [paste]." Specificity makes everything ten times faster.

CHAPTER 23

Pitfalls to avoid

Things that will burn you. Read this twice.

1. Pasting API keys into a public GitHub repo

If you push code to GitHub with an API key inside it, scrapers will find it within minutes and use it. You'll wake up to a $4,000 bill. Always put keys in environment variables. Add .env to your .gitignore. If you do leak one, immediately revoke it in the provider's dashboard — don't just delete the commit.

2. Skipping Supabase Row Level Security

By default, anyone who knows your Supabase URL can read your entire database. RLS (Row Level Security) is the fix and it must be turned on per-table. Every Supabase prompt should include "with Row Level Security enabled and policies that only let users read their own rows."

3. Never committing to Git

The number-one cause of vibe coding tragedies. The AI breaks something subtle. You can't undo. Hours of work lost. Commit at least every 30 minutes when actively building.

4. Asking for too much in one prompt

"Build me Twitter" is not a prompt. "Build a sign-up page using Supabase email auth with a redirect to a dashboard" is a prompt. Slice work into pieces an AI can finish in 2-5 minutes.

5. Trusting AI on security

AI tools will happily build something that works while being trivially hackable. After anything involving logins, payments, or user data, ask: "Audit this code for common security issues — XSS, SQL injection, missing auth checks, exposed secrets, weak validation. List every finding before fixing anything."

6. Believing the AI when it says "I tested this and it works"

Test it yourself. Click the buttons. Try the edge cases. Especially: try with no internet, with an empty database, with invalid input, on a phone. The AI tested its mental model — you have to test reality.

7. Tool-hopping instead of finishing

Every week there's a hot new vibe coding tool. Pick one workflow and ship something. Switch tools between projects, not in the middle.
